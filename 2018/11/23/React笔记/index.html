组件使用

使用步骤

​ 1、在文件夹新建组件文件及其js文件。

​ 2、引入React，并使用jsx语法创建组件内内容

​ 3、在父组件中引入子组件，并使用标签显示。

​ 显示有两种方式：单标签和双标签

组件通信

使用步骤：

​ 1、在父组件中给引入子组件的标签中加入属性名并传值

​ 2、在子组件中使用形参接受传递过来的值

​ 3、使用{props.name}来赋值

状态组件分类

有状态组件

​ 1、相当于一个类，class XY extends Component {} 。

​ 2、继承了Component 的属性可以使用state动态数值。

​ 3、能够使用生命周期函数。

​ 4、可以通过this来接收状态和属性(this.state.XY & this.props.XY)

使用场景：只有在需要管理状态或者需要使用生命周期时使用。

无状态组件

​ 1、是一个函数式，const XY= ( props ) => {}

​ 2、没有state动态数值。

​ 3、没有生命周期函数。

​ 4、可以通过属性实现数据传递(props.XY)

使用场景：基本组件一般都用无状态组件生成，保证页面加载。

生命周期

创建组件时：

​ getDefaultProps()：设置默认属性

​ constructor()：是ES6提供的构造函数，相当于getInitialState()，调用父级super(props)和设置状态state。

​ componentWillMount()：修改状态

​ render()：创建虚拟DOM，更新DOM

​ 加载所有的子组件

​ componentDidMount()：组件渲染完毕（不要更新状态，如果修改就会更新render(),变成死循环）

更新组件时：

​ componententWillReceiveProps(nextProps)：组件发生改变时触发

​ shouldComponentUpdate(nxPro,nxSta)：控制组件是否重新渲染

​ componentWillUpdate(nxPro,nxSta)：进入重新渲染流程

​ runder()

​ 更新与当前相关的子组件

​ componentDidUpdate()：组件渲染完毕

state状态管理

setState() ：专门用来改变state中的状态

同步更新：

异步更新：该函数将接收先前的状态作为第一个参数，将此次更新被应用时的props做为第二个参数：

合并更新：

使用步骤：

​ 1、在父组件中创建state作为管理的库，且只能在拥有继承了Component的下面使用

​ 2、在引入的子组件标签中找到state数据，并使用state中的数据更改子组件状态值

​ 3、如果是静态更改，需要给标签的属性或者标签内使用{this.state.otherState}

​ 如果是动态更改，需要在事件中使用setState({otherState:’状态值’})

事件相关

事件注册

​ 以onClick的驼峰命名形式注册事件，与之前的js事件方法类似。

事件传参

​ 有两种形式传递参数

​ 1、使用ES6语法，让this指向当前class App extends Component

​ onClick={() => this.stateType(‘姓名’)}

​ 2、使用React中bind，让this指向当前class App extends Component

​ onClick={this.stateType.bind(this,’姓名’)}

传递事件

​ 使用步骤：

​ 1、在父组件中使用组件通信的形式传递事件，myclick={this.stateType.bind(this,’miss’)}传递

​ 2、在子组件中给需要绑定的元素加入传递来的属性，onClick={props.myclick}接收

双向绑定

使用步骤：

​ 1、给子组件中绑定onChange={props.changed}

​ 2、在父组件中绑定子组件传递来的属性changed，并使用事件changed={this.inputChange}接收传递来的值

​ 3、在事件中使用形参event接收传递来的参数，并使用setState()来更改状态中的值

​ event.target.value 为当前接收到的值，event.target为目标对象

​ 4、在子组件中使用defaultValue接收父组件赋的值

React中的样式

使用方式分两种：

​ 1、引入样式：

​ 1、直接在对应的子组件文件中创建css，然后在组件中引入使用。

​ 2、在组件中使用className使用样式 ，有两种使用方式，一种一直只用样式类名，还有一种是以对象形式获得样式。

​ 2、行内样式：

​ 1、在render() 中创建const style={}，里面的复合样式用驼峰命名。

​ 2、在给组件标签中添加style={style}，获取样式内容。

if分支

render()：内容发生变化就会从新执行里面内容

使用步骤：

​ 1、在state中声明存放一个if判断你的布尔值。

​ 2、创建更改state状态的事件。

​ 3、在render()内创建存放切换内容的变量和JSX语法中的if判断式，if中的内容存放在【变量名=()】中，并判断state中的状态更改时切换变量中内容。

​ 4、在{}显示内容。

​ 方式一（内容多时）：在return给需要显示的的地方放{showHomeDom}用来显示

​ 方式二（内容少时）：当内容少时可以用三元运算符直接写，就少去步骤三。

for循环

使用步骤：

​ 1、在state中存放需要循环的数组。

​ 2、在需要展示的地方用{}内写上循环语法，return出需要循环的内容。可以用一个唯一的容器包裹需要循环的内容，里面跟dom书写方法一样。

​ 3、要加入Key值，否则会报错。

React路由

基础路由配置

1、安装react-router-dom

​ yarn add react-router-dom

2、配置路由

：跳转地址的导航标签组件

:

​ 1、只找到第一个被location匹配到的就立即停止继续匹配，并且把它渲染出来

​ 2、当前面所有的路由都匹配不上时，会匹配最后一个path="*"的路由，该路由再重定向到404页面。

: 路由渲染的内容

​ path：需要匹配的路由路径

​ runder：渲染的内容

​ exact：只有URL和该的path属性进行精确比对后完全相同该才会被渲染

嵌套路由

match：是实现嵌套路由的对象，当我们在某个页面跳转到它的下一级子页面时，我们不会显示地写出当前页面的路由，而是用match对象的path和url属性

父子路由传参

​ hash: ‘’, //会添加到url里面，形如”#tab1”

​ state: {},//不会添加到url里面

子组件通过this.props.location.search|hash|state获取
